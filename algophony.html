Web-based AI Music Generation Platform

Link

Algophony is an AI-powered music generation platform that offers a streamlined music production experience. It aims to democratize the creative process of music producers by generating individual parts of a song and using them as building blocks for compositions.

Our platform allows professional musicians and enthusiasts alike to describe the instrumental tracks in natural language (e.g. calm and soothing electric guitar) and generate high quality audio using CVAE and HifiGAN. Users can either use the generated loops in their own audio workstations or edit these loops onsite. After editing and mixing, they can directly export and share the music they created!

Data Processing

Our training data consists of ~30k audio samples sourced from music sample pack sites like Splice. By using the same samples that many of the industry producers use, we aim to deliver professional sounds! We ran pre-trained classification models from Essentia on these data to gather information of their instrument type, genre, mood and theme (as visualized below). Then, we generated the mel-spectrograms from this data using Short-Time Fourier Transform (STFT) for the visual representation. These spectrograms are the input data in our CVAE training steps.

Generation Model: CVAE

The main idea is that our model is trained to generate new spectrograms based on existing data. A regular VAE with matching layers in its encoder and decoder can learn data patterns effectively. Once trained, it can generate new spectrograms by randomly sampling from our model's normally distributed latent space. However, to generate specific sounds, the model needs to learn the nuances of different categories of sounds in the training stage. We approach this by introducing a multi-class label encoding into our input, thus making the model conditional on the label it's been given, which is CVAE.

Here's how we implemented this encoder-decoder architecture using PyTorch.
The learning objective of our model is to minimize the loss function defined by a balance of reconstruction error (calculated by mean squared error) and KL-Divergence. Using the loss function, we improve our latent space parameters with ADAM optimization algorithm in each iteration.

class CVAE(nn.Module):
    def __init__(self, feature_size, latent_size, class_size):
        super(CVAE, self).__init__()
        self.feature_size = feature_size
        self.latent_size = latent_size
        self.class_size = class_size
        # encoder structure
        self.fc1  = nn.Linear(feature_size + class_size, h_latent_size)
        self.fc21 = nn.Linear(h_latent_size, latent_size)
        self.fc22 = nn.Linear(h_latent_size, latent_size)
        # decoder structure
        self.fc3 = nn.Linear(latent_size + class_size, h_latent_size)
        self.fc4 = nn.Linear(h_latent_size, feature_size)
        self.elu = nn.ELU()
        self.sigmoid = nn.Sigmoid()
    def encode(self, x, c): # Q(z|x, c)
        inputs = torch.cat([x, c], 1) # feature_size + class_size
        h1 = self.elu(self.fc1(inputs))
        z_mu = self.fc21(h1)
        z_var = self.fc22(h1)
        return z_mu, z_var
	
    # reparameterization trick to allow for backpropagation
    def reparameterize(self, mu, logvar):
        std = torch.exp(0.5*logvar)
        eps = torch.randn_like(std)
        return mu + eps*std

    def decode(self, z, c): # P(x|z, c)
        inputs = torch.cat([z, c], 1) # latent_size + class_size
        h3 = self.elu(self.fc3(inputs))
        return self.sigmoid(self.fc4(h3))

    def forward(self, x, c):
        mu, logvar = self.encode(x.view(-1, 3*width*height), c)
        z = self.reparameterize(mu, logvar)
        return self.decode(z, c), mu, logvar

    def generate(self, c, num_samples=1):
        with torch.no_grad():
            z = torch.randn(num_samples, self.latent_size).to(c.device)
            x_generated = self.decode(z, c)
        return x_generated

Audio Conversion with Hifi-GAN

The CVAE model outputs spectrogram that represents new audio. To make the conversion back to audio, we employ Hifi-GAN model for speech synthesis. Compared to alternative methods like Griffin-Lim, this approach reduces the introduction of lossy artifacts in the conversion process and help generate better quality sounds.
